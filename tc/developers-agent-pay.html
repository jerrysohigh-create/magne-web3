目標整合流程

下圖說明目標端到端 Agent Pay 工作流程。處於開發或驗證階段的元件以黃色顯示。

1

代理服務調用 → HTTP 402 回應

代理服務收到請求並返回 HTTP 402 Payment Required（含支付元數據）。此流程正在積極開發中，尚未可用於生產整合。

2

瀏覽器錢包授權

客戶端啟動錢包授權以支付金額（以 mMHA 計算）。瀏覽器擴展整合正在開發中。當前演示使用直接錢包交互，無瀏覽器擴展自動化。

3

M Hash L2 代幣轉帳

授權支付作為 mMHA（測試網／演示 ERC-20）在 M Hash L2 測試網上轉移至促進者地址。此步驟作為已披露之測試網階段演示架構的一部分進行說明。

4

促進者驗證 → AI 任務收據

促進者驗證支付並在鏈上觸發 AI 任務收據創建。促進者驗證及結算邏輯正在開發中。

⚠️ 開發狀態聲明

上述完整的 HTTP 402 支付流程、瀏覽器錢包授權及促進者驗證／結算邏輯仍處於開發中。它們已作為目標架構和測試網演示元件公開披露，而非作為生產就緒的整合。正式的 x402 協議相容性正在驗證中。

測試環境

Agent Pay 演示整合的可用測試環境。

# M Hash L2 測試網配置 chainId: 20250827 網絡名稱： M Hash L2 測試網 RPC 端點： https://l2-rpc.testnet.magicalhash.com 瀏覽器： https://l2-explorer.testnet.magicalhash.com 資產： mMHA（ERC20，測試網） 狀態： 測試網階段開發者演示

🔧 僅限開發環境

M Hash L2 測試網是一個開發和測試環境，不是主網。測試網上的 mMHA 代幣沒有貨幣價值，不代表生產結算資產。

x402 相容性

正式的 x402 協議相容性正在驗證中。HTTP 402 支付流程正在設計中，以 x402 相容性為目標。

⏳ 驗證中

x402 是一個新興的 HTTP 支付頭標準。MAGNE Agent Pay 正在設計中以實現 x402 相容性，但此整合尚未經過正式驗證或認證。請勿將 x402 相容性作為已完成的特性進行宣傳。

安全行動端授權、M Hash L2 測試網演示與 AI 任務收據記錄，服務於正在形成的代理支付工作流程。
