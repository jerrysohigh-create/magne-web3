測試網階段開發者演示

AI 代理能思考。
仍然無法安全地支付。

MAGNE Agent Pay 是一個 x402 相容的 AI 代理支付開發者演示 — 結合錢包授權、促進者驗證與 M Hash L2 上的不可篡改 AI 任務收據。

→ 查看 Agent Pay 演示 閱讀開發文檔
核心問題

每個代理經濟體都需要支付軌道。目前尚不存在。

AI 代理可以執行任務，卻無法獲得可靠的支付。傳統 API 返回 401。傳統支付缺乏可審計性。

🔐
01 — 授權

無安全授權機制

代理可以觸發任務，但支付仍需要可信的中間人。目前沒有密碼學可驗證的方式讓代理請求並接收授權。

💸
02 — 支付軌道

無代理支付流程

API 和模型服務需要可編程呼叫的按用量付費結算。現有支付 API 專為人類和瀏覽器設計。

📋
03 — 可審計性

無可驗證收據

AI 任務需要可審計的記錄：誰發起、誰支付、結算了什麼、返回了什麼。沒有收據就沒有信任。

工作原理

從任務請求到鏈上收據 — 毫秒級完成。

專為機器對機器結算設計的完整支付流程，支持通過 MAGNE Phone 或外部錢包進行人工覆寫。

步驟 01

任務請求

代理調用 API 並附上任務參數。API 返回 HTTP 402 Payment Required，含支付元數據。

→
步驟 02

402 響應

支付金額、收款人地址、鏈 ID 和截止時間嵌入 402 響應頭部。

→
步驟 03

授權

用戶通過 MAGNE Phone Secure Element 或外部錢包授權。授權在鏈下但被記錄。

→
步驟 04

結算

促進者在 M Hash L2 上驗證支付。mMHA 或支持的資產原子化轉移。

→
步驟 05

AI 任務收據

不可篡改的收據在 M Hash L2 上鑄造。包含 receiptId、taskId、雙方、金額、代幣、paymentTxHash、時間戳。

收據層

每筆支付都會留下痕跡。讓它不可篡改。

AI 任務收據是存儲在 M Hash L2 上的 EVM 事件日誌。它們不可篡改、可編程、任何一方（包括代理本身）都可驗證。

event AITaskReceiptCreated
receiptId bytes32 // 唯一收據標識符，也是交易哈希
taskId string // 代理生成的任務標識符
user address // 發起任務的 EOA 或合約
provider address // 驗證支付的促進者地址
amount uint256 // 最小單位金額（如 wei）
token address // mMHA 或其他支持的結算代幣
paymentTxHash bytes32 // 鏈上支付交易哈希
timestamp uint256 // 結算時的區塊時間戳
授權層

人類攜帶的最安全設備就是手機。用這一點。

MAGNE.AI Phone 整合了 Secure Element（eSE + TEE），可在不暴露私鑰的情況下簽署授權請求。代理獲得支付。用戶保持控制。

🔐

Secure Element (eSE)

硬件級密鑰存儲。私鑰永不離開安全晶片。授權由密碼學強制執行，而非僅靠策略。

📱

TEE 運行時

可信執行環境隔離運行。授權邏輯在隔離環境中執行，主系統和任何應用都無法訪問。

🔑

x402 相容

MAGNE Phone 使用標準 HTTP 402 協議。任何 x402 相容 API 可直接與其交互，無需自定義 SDK。

開發者

你的 API + 一份 SKILL.md = 一個代理服務。

為任何 API 端點附上一份 SKILL.md。代理發現它、調用它、為它付費 — 編程化地，無需人工干預。
