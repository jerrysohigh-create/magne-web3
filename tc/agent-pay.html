核心問題

AI 代理遭遇支付壁壘。

當前基礎設施讓代理能思考，但無法可靠地交易。

401 無安全授權

代理可以觸發。用戶無法驗證。

代理可以觸發任務，但支付仍需要可信的人類逐步授權。在新興代理支付場景中，如何形成可驗證且可控的授權路徑，仍是重要實作課題。

0 無代理支付軌道

服務存在。代理無法支付。

代理支付軌道仍在形成中。安全行動端授權和可驗證任務收據仍是開放的實施挑戰。

0 無可驗證收據

任務執行。無法證明。

AI 任務需要可審計的記錄：誰發起、誰支付、結算了什麼、返回了什麼。部分現有工作流程未能提供一致、可獨立查驗的鏈上記錄。

鏈上記錄查驗

AI 任務收據層

在已披露的測試網演示中，AI 任務收據用於記錄與查驗相關鏈上資訊，展示未來代理支付工作流程中的可驗證收據方向。

為什麼收據重要

AI 任務收據是可驗證的證明，AI 任務收據可用於記錄已披露演示中的支付參考資料、工作資訊與結果雜湊等鏈上資料。任何一方，包括 AI 代理、使用者、審計方或智能合約，均可透過鏈上資料獨立查驗相關記錄。

收據被設計為可編程的：代理可以在後續交易中引用它們、爭議解決合約可以讀取它們、合規系統可以在不需要可信中介的情況下審計它們。

合約事件

當收據最終確認時，AITaskReceiptCreated event 會在 M Hash L2 上觸發：

開發者整合

以 SKILL.md 描述代理可調用的服務

SKILL.md 定義了面向代理的合約。代理讀取它；服務履行它。不需要專有 SDK。

// SKILL.md — AI Agent Service Contract { "skill": "錢包風險分析", "version": "1.0", "provider": "MAGNE-Agent-001", "price": "0.01 mMHA", "chain": "mhash-l2-testnet", "chainId": 20250827, "paymentToken": "mMHA (ERC20)", "facilitator": "https://api.magne.ai/facilitator", "receiptContract": "0xa0Fe7AD300152eeDC4ca0c17514A1001D6afA408", "x402Compatibility": "under-validation", "endpoints": { "analyze": "POST /api/v1/wallet-risk", "verify": "GET /api/v1/receipt/:id" } } // 代理流程： 1. POST /api/v1/wallet-risk → 402 + 支付元資料 2. mMHA.transfer(recipient, amount) on M Hash L2 3. // 目標流程示例；尚未完成公開端到端驗證 // Facilitator.strictVerify(paymentTxHash, 任務ID, amount) 4.
