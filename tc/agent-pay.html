核心問題

AI 代理遭遇支付壁壘。

當前基礎設施讓代理能思考，但無法可靠地交易。

401 無安全授權

代理可以觸發。用戶無法驗證。

代理可以觸發任務，但支付仍需要可信的人類逐步授權。目前沒有密碼學可驗證的授權路徑。

0 無代理支付軌道

服務存在。代理無法支付。

代理支付軌道正在興起。安全行動端授權和可驗證任務收據仍是開放的實施挑戰。

0 無可驗證收據

任務執行。無法證明。

AI 任務需要可審計的記錄：誰發起、誰支付、結算了什麼、返回了什麼。現有工作流程不留下可追蹤記錄。

On-Chain Verification

AI Task 收據層

每筆支付都在 M Hash L2 上創建不可篡改、可編程的收據。

為什麼收據重要

AI 任務收據是可驗證的證明，證明支付已發生、任務已請求、結果已返回。任何一方，包括 AI 代理、使用者、審計方或智能合約，均可透過鏈上資料獨立查驗相關記錄。

收據被設計為可編程的：代理可以在後續交易中引用它們、爭議解決合約可以讀取它們、合規系統可以在不需要可信中介的情況下審計它們。

合約事件

The AITaskReceiptCreated event is emitted on M Hash L2 when a receipt is finalized:

事件 AITaskReceiptCreated(
收據IDbytes32
任務IDstring
useraddress
provideraddress
amountuint256
tokenaddress
paymentTxHashbytes32
timestampuint256
)
Developer Integration

以 SKILL.md 描述代理可調用的服務

SKILL.md 定義了面向代理的合約。代理讀取它；服務履行它。不需要專有 SDK。

// SKILL.md — AI Agent Service Contract { "skill": "錢包風險分析", "version": "1.0", "provider": "MAGNE-Agent-001", "price": "0.01 mMHA", "chain": "mhash-l2-testnet", "chainId": 20250827, "paymentToken": "mMHA (ERC20)", "facilitator": "https://api.magne.ai/facilitator", "receiptContract": "0xa0Fe7AD300152eeDC4ca0c17514A1001D6afA408", "x402Compatible": true, "endpoints": { "analyze": "POST /api/v1/wallet-risk", "verify": "GET /api/v1/receipt/:id" } } // Agent flow: 1. POST /api/v1/wallet-risk → 402 + payment metadata 2. mMHA.transfer(recipient, amount) on M Hash L2 3. Facilitator.strictVerify(paymentTxHash, 任務ID, amount) 4. AI Task Receipt event → on-chain proof

Ecosystem Position

MAGNE 與其他代理支付方案的比較
