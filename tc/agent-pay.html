核心問題

AI 代理遭遇支付壁壘。

當前基礎設施讓代理能思考，但無法可靠地交易。

401 無安全授權

代理可以觸發。用戶無法驗證。

代理可以觸發任務，但支付仍需要可信的人類逐步授權。在新興代理支付場景中，如何形成可驗證且可控的授權路徑，仍是重要實作課題。

0 無代理支付軌道

服務存在。代理無法支付。

代理支付軌道仍在形成中。安全行動端授權和可驗證任務收據仍是開放的實施挑戰。

0 無可驗證收據

任務執行。無法證明。

AI 任務需要可審計的記錄：誰發起、誰支付、結算了什麼、返回了什麼。部分現有工作流程未能提供一致、可獨立查驗的鏈上記錄。

鏈上驗證

AI 任務收據層

M Hash L2 上的 AI 任務收據記錄用於任務驗證。

為什麼收據很重要

AI 任務收據是支付完成、任務請求、結果返回的可驗證證明。任何一方——代理、用戶、審計員或智能合約——都可以在鏈上獨立驗證完整歷史。

收據被設計為可編程的：代理可以在後續交易中引用它們，爭議解決合約可以讀取它們，合規系統可以在不需要可信中介的情況下進行審計。

合約事件

當收據最終確定時，AITaskReceiptCreated 事件會在 M Hash L2 上發出：

event AITaskReceiptCreated(
receiptIdbytes32
taskIdstring
useraddress
provideraddress
amountuint256
tokenaddress
paymentTxHashbytes32
timestampuint256
)
開發者整合

你的代碼 + SKILL.md = Aapp

SKILL.md 定義了面向代理的合約。代理讀取它，服務履行它。無需專有 SDK。

// SKILL.md — AI Agent Service Contract { "skill": "wallet-risk-analysis", "version": "1.0", "provider": "MAGNE-Agent-001", "price": "0.01 mMHA", "chain": "mhash-l2-testnet", "chainId": 20250827, "paymentToken": "mMHA (ERC20)", "facilitator": "https://api.magne.ai/facilitator", "receiptContract": "0xa0Fe7AD300152eeDC4ca0c17514A1001D6afA408", "x402Compatibility": "under-validation" } // Agent flow: 1. POST /api/v1/wallet-risk → 402 + payment metadata 2. mMHA.transfer(recipient, amount) on M Hash L2 3. Facilitator.strictVerify(paymentTxHash, taskId, amount) 4. AI Task Receipt event → on-chain proof
