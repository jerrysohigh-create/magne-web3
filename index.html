Agents can think.
They still cannot safely pay.

MAGNE Agent Pay is an x402-compatible developer demo for AI agent payment — combining wallet authorization, facilitator verification, and immutable AI Task Receipts on M Hash L2.

The Problem

Every agent economy needs payment rails. None exist yet.

AI agents can execute tasks. They cannot receive reliable payment. Traditional APIs return 401. Traditional payments lack auditability.

🔐
01 — Authorization

No Secure Authorization

Agents can trigger tasks, but payments still require trusted intermediaries. There's no cryptographically verifiable way for an agent to request and receive authorization.

💸
02 — Payment Rail

No Agent Payment Flow

APIs and model services need pay-per-use settlement that agents can call programmatically. Existing payment APIs are designed for humans with browsers.

📋
03 — Auditability

No Verifiable Receipts

AI tasks need auditable records: who requested, who paid, what settled, and what was returned. Without receipts, there's no trust between parties.

How It Works

From task request to on-chain receipt — in milliseconds.

A complete payment flow designed for machine-to-machine settlement, with human override capability through MAGNE Phone or external wallet.

STEP 01

Task Request

Agent calls the API with task parameters. API returns HTTP 402 Payment Required with payment metadata.

→
STEP 02

402 Response

Payment amount, recipient address, chain ID, and deadline embedded in the 402 response headers.

→
STEP 03

Authorization

User authorizes via MAGNE Phone Secure Element or external wallet. Authorization is off-chain but logged.

→
STEP 04

Settlement

Facilitator verifies payment on M Hash L2. mMHA or supported asset transfers execute atomically.

→
STEP 05

AI Task Receipt

Immutable receipt minted on M Hash L2. Contains receiptId, taskId, parties, amount, token, paymentTxHash, timestamp.

Receipt Layer

Every payment leaves a trace. Make it immutable.

AI Task Receipts are EVM event logs stored on M Hash L2. They are immutable, programmable, and verifiable by any party — including the agent itself.

event AITaskReceiptCreated
receiptId bytes32 // Unique receipt identifier, also the tx hash
taskId string // Agent-generated task identifier
user address //EOA or contract that initiated the task
provider address // Facilitator address that verified the payment
amount uint256 // Amount in smallest unit (e.g., wei)
token address // mMHA or other supported settlement token
paymentTxHash bytes32 // On-chain payment transaction hash
timestamp uint256 // Block timestamp at settlement
Authorization Layer

Phones are the most secure devices humans carry. Use that.

MAGNE.AI Phone integrates a Secure Element (eSE + TEE) that can sign authorization requests without exposing private keys. The agent gets its payment. The user stays in control.

🔐

Secure Element (eSE)

Hardware-level key storage. Private keys never leave the secure chip. Authorization is cryptographically enforced, not just policy-based.

📱

TEE Runtime

Trusted Execution Environment隔离执行环境. Authorization logic runs in an isolated environment that cannot be accessed by the main OS or any app.

🔑

x402 Compatible

MAGNE Phone speaks the standard HTTP 402 protocol. Any x402-compatible API can interact with it directly — no custom SDK required.
