Target Integration Flow

The diagram below illustrates the target end-to-end Agent Pay workflow. Components currently in development or under validation are shown in yellow.

1

Agent Service Call → HTTP 402 Response

Agent service receives a request and returns HTTP 402 Payment Required with payment metadata. This flow is in active development and not yet available for production integration.

2

Browser Wallet Authorization

Client initiates wallet authorization for the payment amount in mMHA. Browser extension integration is in development. Current demo uses direct wallet interaction without browser extension automation.

3

Token Transfer on M Hash L2

Authorized payment is transferred as mMHA (testnet/demo ERC-20) on M Hash L2 testnet to the facilitator address. This step has been demonstrated in the testnet-stage evidence.

4

Facilitator Verify → AI Task Receipt

Facilitator verifies payment and triggers AI Task Receipt creation on-chain. Facilitator verification and settlement logic is in development.

⚠ Development Status Disclaimer

The full HTTP 402 payment flow, browser wallet authorization, and Facilitator verify/settle logic described above remain in development. They have been publicly disclosed as target architecture and testnet demonstration components — not as production-ready integrations. Formal x402 protocol compatibility is under validation.

x402 Compatibility

Formal x402 protocol compatibility is currently under validation. The HTTP 402 payment flow is being designed with x402 compatibility as a target.

⏳ Under Validation

x402 is an emerging HTTP payment header standard. MAGNE Agent Pay is being designed for x402 compatibility, but this integration has not been formally validated or certified. Do not represent x402 compatibility as a completed feature.
